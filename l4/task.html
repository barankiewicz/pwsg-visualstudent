WPF2 - Visual Student

Zadanie polega na napisaniu prostego edytora plików C#, który umożliwia zbudowanie całego projektu oraz jego uruchomienie.

Część laboratoryjna

  • Aplikacja przykładowa
  • Główne okno:
    • rozmiar początkowy: 800px x 600px, rozmiar minimalny 500px x 500px
    • okno pojawia się na środku ekranu
    • główne okno zawiera:
      • menu na górze
      • edytor tekstu - kontrolka składająca się z wielu zakładek, umożliwiająca jednoczesną edycję wielu plików C# (rozszerzenie .cs)
      • drzewo projektu - drzewo przedstawiająca całą strukturę katalogową otwartego projektu, umożliwiające otwarcie pliku do edycji poprzez podwójne kliknięcie
    • pomiędzy edytorem tekstu, a drzewem projektu znajduje się splitter, który umożliwia zmianę szerokości każdej z tych części aplikacji
  • Menu:
    • File
      • New - tworzy nową zakładkę z pustym edytorem tekstowym
      • Open file - umożliwia otwarcie pliku z rozszerzeniem .cs (ustawiony odpowiedni filtr) oraz późniejszą jego edycję w edytorze
      • Open project - umożliwia otwarcie projektu C# oraz wczytanie jego struktury do drzewa znajdującego się po prawej stronie aplikacji (warunkiem określającym czy dany folder zawiera poprawny projekt C# jest istnienie pliku z rozszerzeniem .csproj w katalogu)
      • Exit - zamyka aplikację
    • About - wyświetla krótką informację o aplikacji
  • Edytor tekstu:
    • podzielony na zakładki (jedna zakładka = jeden otwarty plik)
    • tytuł każdej zakładki to nazwa pliku, a w przypadku utworzenia nowego pliku tytuł zakładki to 'New file'
    • każda zakładka zawiera prosty edytor umożliwiający edycje pliku
  • Drzewo projektu
    • przedstawia strukturę katalogową otwartego projektu
    • wyświetla pliki z rozszerzeniem .cs
    • podwójne kliknięcie na element reprezentujący plik .cs powoduje otwarcie tego pliku do edycji
    • w przypadku próby wczytania nieprawidłowego projektu (katalog, który nie zawiera pliku .csproj) wyświetlony zostanie komunikat informujący o błędzie
  • Wskazówki:
    • WindowStartupLocation, Window.MinHeight, Window.Height, Window.MinWidth, Window.Width
    • Menu, MenuItem
    • RichTextBox
    • ListBox, ListView.ItemTemplate, DataTemplate
    • TreeView
    • FolderBrowserDialog (System.Windows.Forms)
  • Punktacja:
    • menu + layout okna + 'New' + 'Open file' + edytor tekstu z zakładkami zawierającymi otwarte lub nowo utworzone pliki - 4 punkty
    • menu + layout okna + 'Open project' + drzewo projektu - 4 punkty

Część domowa

  • Aplikacja przykładowa
  • Spełnienie wymagań z części pierwszej jest warunkiem koniecznym do oceny części drugiej.
  • Główne okno:
    • główne okno podzielone na 4 główne sekcje
    • sekcja budowania projektu, znajdująca się pod menu, zawiera combo box umożliwiającego wybór pomiędzy samym zbudowaniem projektu, a zbudowaniem oraz uruchomieniem wygenerowanego pliku exe
    • sekcja po prawej stronie zawiera drzewo projektu
    • górna sekcja po lewej stronie zawiera zakładki z otwartymi plikami C#
    • dolna sekcja po lewej stronie zawiera output z procesu kompilacji projektu oraz listę znalezionych błędów
  • Menu:
    • dodany element menu 'Save' oraz 'Save as', które umożliwiają zapisanie edytowanego pliku
    • nowy element menu 'Plugins' zawierający listę wczytanych pluginów z plików dll
    • każdy plugin może zostać włączony lub wyłączony poprzez kliknięcie na element menu w zakładce 'Plugins'
    • jednocześnie może być włączonych kilka pluginów
  • Skróty klawiszowe:
    • 'Ctrl + N' - tworzy nowy plik
    • 'Ctrl + O' - otwiera plik
    • 'Ctrl + S' - zapisuje plik
    • 'F5' - uruchamia kompilację (plus ewentualnie uruchamia wygenerowany plik exe)
  • Sekcja budowania projektu:
    • combo box, który umożliwia wybór czy chcemy tylko zbudować projekt czy dodatkowo uruchomić wygenerowany plik exe
    • przycisk 'Execute' umożliwiający uruchomienie procesu kompilacji
  • Edytor tekstowy:
    • użytkownik ma możliwość zamknięcia zakładki poprzez kliknięcie przycisku 'X'
    • jeśli dokument został edytowany, to w tytule zakładki dodany zostaje znak '*'
    • jeśli dokument został edytowany i użytkownik chce go zamknąć, wyświetlany jest odpowiedni komunikat (patrz aplikacja przykładowa)
  • Output oraz lista błędów:
    • sekcja zawiera dwie zakładki
    • w zakładce 'Output' wyświetlony zostaje cały output, który generuje BuildManager w procesie kompilacji projektu
    • w zakładce 'Error List' wyświetlone są ewentualne błędy, które wystąpiły podczas kompilacji - należy stworzyć odpowiedni DataTemplate
    • po najechaniu na nazwę każdej z zakładki aktywuje się animacja (patrz aplikacja przykładowa)
  • System pluginów:
    • należy zaimplementować dwa pluginy: pierwszy kolorujący na niebiesko słowa kluczowe (public, private, get, set), drugi kolorujący na czerwono i pogrubiający słowa kluczowe (using, if, else, for, foreach)
    • system pluginów powinien umożliwiać dodanie nowego pluginu bez konieczności ponownej kompilacji projektu - poprzez dodanie odpowiedniego pliku dll do katalogu z plikiem exe
    • każdy plugin powinien być zaimplementowany jako class library (generowany do pliku dll), który implementuje poniższy interfejs: 
      public interface IPlugin
{
    string Name { get; }
    void Do(RichTextBox richTextBox);
}
    • system pluginów powinien być zaimplementowany w sposób, który umożliwia wymianę pluginów pomiędzy studentami
    • poniżej przedstawiono przykład implementacji pluginu 
      public class SyntaxHighlightPlugin : IPlugin
{
    public string Name
    {
        get
        {
            return "Sample Plugin";
        }
    }

    public void Do(RichTextBox richTextBox)
    {
        // DO SOMETHING
    }
}
  • Wskazówki:
    • NotifyPropertyChanged
    • ComboBox, ComboBoxItem
    • TabControl, TabItem
    • EventTrigger, DoubleAnimation
    • InputBindings, KeyBinding, Command
    • BuildManager, BuildParameters, ConsoleLogger
    • Build Path in Project properties
  • Punktacja:
    • proces kompilacji projektu i uruchamiania wygenerowanego pliku exe - 2 punkty
    • wyświetlanie outputu z kompilacji oraz listy błędów - 2 punkty
    • zamykanie zakładek z plikami C# - 1 punkt
    • sygnalizacja zmian w pliku (znak '*') oraz zapisywanie plików - 1 punkt
    • animacja zakładek 'Output' oraz 'Error List' - 1 punkt
    • skróty klawiszowe - 1 punkt
    • system pluginów - 4 punkty (bez zaimplementowanego systemu, każdy plugin jest wart 0.5pkt)
